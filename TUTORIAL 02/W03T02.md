3. What is a raw type? Why is a raw type unsafe? Why is the raw type allowed in Java?

    ```
    Raw type is  a generic class/interface used without specifying a concrete type/without a type parameter. 

    Raw type is unsafe because it can cause unchecked runtime error. For example , if we are comparing object1 , which is a String , with object2 ,which is an Integer, it will cause runtime error. 

    Raw types can be useful when interfacing with non-generic legacy code.Thus, the generic code can bebackward-compatible with the legacy code that uses raw types.
    ```

4. What is erasure? Why are Java generics implements using erasure?
   
   ```
   Type erasure is when the compiler uses the generic type information to compile the code, but erases it afterwards.

   Java generics are impelmented using erasure to uphold the property of "parametricity" of a data type.For example , I have a type "List" parameterised over component type T. i.e. List<T>. That type is a proposition that this List type works identically for any type T. The fact that T is an abstract, unbounded type parameter means that we know nothing about this type, therefore are prevented from doing anything special for special cases of T.

   Erasure prevents some means of violating parametricity, thus eliminating possibilities of incorrect programs.

   ```
